GitHub Settings
Connect once — push any summary directly to your repo, auto-sorted by domain.
🔑 Personal Access Token
Generate a token at github.com → Settings → Developer settings → Personal access tokens → Fine-grained. Grant Contents: Read & Write on your target repo only.

🔒 Security: Your token is held in sessionStorage only — it is never written to disk or localStorage, and clears automatically when you close the tab. You will need to re-enter it each session.

📂 File Path — You Choose Every Time
When you push a summary, a modal will ask you to confirm (or edit) the file path and commit message before anything is sent. Nothing is pushed without your explicit review.

The suggested path is pre-filled as {domain}/{paper-slug}.md — e.g. machine-learning/attention-is-all-you-need.md — but you can type any path you want before confirming.
🐙 Push to GitHub
Review the file path before pushing. The file will be created or updated in your repo.
